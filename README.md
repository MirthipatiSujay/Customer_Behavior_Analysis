# Customer_Behavior_Analysis


## Overview

This project demonstrates an end-to-end data analytics workflow using Python, MySQL, and Power BI. It covers data loading, exploratory data analysis (EDA), data cleaning, SQL-based analysis, and interactive dashboard creation to generate meaningful business insights.

The project showcases practical data analysis skills, SQL querying, and data visualization techniques commonly used in real-world analytics projects.

---

## Dataset

The dataset contains structured business data used for analysis and reporting.

The analysis includes:

* Data quality assessment
* Handling missing values and duplicates
* Exploratory Data Analysis (EDA)
* SQL-based data querying
* Interactive dashboard visualization

## Project Workflow

### 1. Data Loading

* Imported the dataset into Python.
* Inspected data structure, data types, and summary statistics.

### 2. Exploratory Data Analysis (EDA)

* Analyzed distributions and trends.
* Identified missing values and duplicate records.
* Explored relationships between variables.
* Created visualizations for better understanding.

### 3. Data Cleaning

* Removed duplicate records.
* Handled missing values.
* Corrected inconsistent data formats.
* Prepared a clean dataset for analysis.


### 4. SQL Analysis

After cleaning the dataset, it was imported into MySQL Server for business analysis. SQL queries were written to answer key business questions and uncover customer purchasing patterns.

The analysis includes:

* Comparing total revenue generated by male and female customers.
* Identifying customers who received discounts but still spent above the average purchase amount.
* Finding the top-rated products based on average customer review ratings.
* Comparing average purchase amounts across different shipping methods.
* Analyzing customer spending behavior based on subscription status.
* Calculating the percentage of discounted purchases for each product.
* Segmenting customers into **New**, **Returning**, and **Loyal** groups using previous purchase history.
* Identifying the top-selling products within each product category using window functions.
* Evaluating the relationship between repeat purchases and customer subscriptions.
* Measuring revenue contribution across different age groups.

The SQL analysis demonstrates the use of aggregate functions, filtering, grouping, subqueries, Common Table Expressions (CTEs), window functions, conditional logic, and business-oriented reporting.


### 5. Power BI Dashboard

* Connected Power BI to the MySQL database.
* Created an interactive dashboard using charts, KPIs, slicers, and filters.
* Built visual reports to support data-driven decision making.

---

## Dashboard

An interactive Power BI dashboard was developed to visualize customer shopping behavior and business performance. The dashboard includes:

* KPI cards displaying total customers, average purchase amount, and average review rating.
* Revenue by product category.
* Sales distribution by product category.
* Revenue generated across different age groups.
* Customer distribution by age group.
* Customer subscription status breakdown using a donut chart.
* Interactive slicers for filtering data by gender, category, subscription status, and age group.

The dashboard enables users to explore customer demographics, purchasing behavior, revenue trends, and category performance through dynamic and interactive visualizations.

## How to Run

1. Clone the repository.
2. Install the required Python libraries:

   ```
   pip install pandas sqlalchemy pymysql
   ```
3. Open the Python file and run the code.
4. Import the cleaned dataset into MySQL Server.
5. Execute the SQL queries.
6. Open the Power BI (.pbix) file.
7. Refresh the data connection to view the latest dashboard.
